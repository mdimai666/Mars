# JoinNode

## Описание

**Join Node** — это узел, который объединяет несколько входящих сообщений в одно. Он работает как сборщик данных: ждёт сообщения с разных входов или определённое количество сообщений, а затем формирует из них массив и передаёт дальше.

---

## Когда использовать

Используйте Join Node, когда нужно:
- Объединить данные из нескольких источников перед обработкой
- Дождаться завершения нескольких параллельных процессов
- Собрать группу сообщений за определённое время
- Получить фиксированное количество сообщений перед выполнением следующего шага

---

## Режимы работы

Узел поддерживает три режима, которые выбираются в свойстве **Mode**:

### 1. InputAggregation (ожидание всех входов)

Ждёт, пока на **каждый** входной порт придёт хотя бы одно сообщение. После этого все полученные данные собираются в массив и отправляются в выходной порт `on join`.

Если какой-то порт не получил сообщение в течение заданного таймаута, накопленные данные отправятся в порт `on InputAggregation timeout`, а ожидание для этой задачи завершится.

**Настройка:** `InputAggregationTimeoutSeconds` — время ожидания в секундах (по умолчанию 15).

### 2. CountAggregation (сбор по количеству)

Собирает указанное количество сообщений, **независимо от того, с какого входа** они пришли. Как только набрано нужное число сообщений, все они отправляются массивом.

**Настройка:** `MessageCount` — количество сообщений для сбора (по умолчанию 5).

### 3. TimeAggregation (сбор по времени)

Собирает все сообщения, пришедшие за указанный интервал времени, и отправляет их единым массивом после окончания интервала.

**Настройка:** `AggregationTimeSeconds` — длительность сбора в секундах (по умолчанию 5).

---

## Важные замечания

- В режиме `InputAggregation` каждая задача отслеживается отдельно — разные выполнения не смешиваются
- При таймауте в режиме `InputAggregation` частично собранные данные не теряются, а отправляются в специальный выходной порт
- В режимах `CountAggregation` и `TimeAggregation` сообщения обрабатываются в порядке поступления
