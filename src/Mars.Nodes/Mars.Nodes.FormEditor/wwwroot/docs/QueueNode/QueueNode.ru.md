# Узел «Очередь» (QueueNode)

Узел предназначен для структурированной и контролируемой обработки потока сообщений. Он накапливает входящие данные и отдает их на дальнейшую обработку строго дозированно, дожидаясь завершения предыдущих операций.

## Основные возможности

* **Гибкие режимы извлечения**: 
  * **FIFO** (First In, First Out) – обрабатывается тот элемент, который пришел первым (стандартная очередь).
  * **LIFO** (Last In, First Out) – обрабатывается тот элемент, который пришел последним (стек).
* **Контроль нагрузки**: Ограничение максимального количества одновременно выполняемых задач (`MaxTask`). Это защищает следующие узлы или внешние системы от перегрузки.

## Настройки узла

| Параметр | Описание | Значение по умолчанию |
| :--- | :--- | :--- |
| **Режим (Mode)** | Порядок извлечения элементов: `FIFO` или `LIFO`. | `FIFO` |
| **Макс. задач (MaxTask)** | Максимальное количество элементов, которые могут обрабатываться одновременно. | `1` |

## Как использовать

1. **Добавление данных**: Просто отправьте сообщение на вход узла. Оно будет автоматически добавлено в очередь.
2. **Автоматическая обработка**: Узел самостоятельно передаст следующий элемент на выход, как только количество активных задач станет меньше значения `MaxTask`.
3. **Завершение**: Когда все элементы обработаны и очередь пуста, узел сообщит о полном завершении цикла.
