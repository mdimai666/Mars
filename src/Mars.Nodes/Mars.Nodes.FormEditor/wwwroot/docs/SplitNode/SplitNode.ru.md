# SplitNode

Узел предназначен для декомпозиции (разделения) сложных или составных данных на отдельные элементы. Он принимает одно сообщение на вход и генерирует серию сообщений на выход, вызывая следующий шаг обработки для каждой полученной части.

## Как это работает

Узел автоматически определяет тип входящих данных и применяет соответствующую стратегию разделения:

1. **Строки**: 
   - Разделяет текст на подстроки, используя заданный **Разделитель**.
   - Если поле разделителя оставить **пустым**, текст будет разделен посимвольно (каждая буква или символ станут отдельным сообщением).
2. **Массивы и списки** (IEnumerable): 
   - Проходит по каждому элементу коллекции и отправляет его на выход по отдельности.
3. **Сложные объекты**: 
   - Анализирует объект и создает отдельное сообщение для каждого его публичного свойства. На выход передается структура, содержащая имя свойства (`PropertyName`) и его значение (`Value`).
